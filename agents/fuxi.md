                      │
    坎 (数据流) ───┼─── 离 (性能资源)

## 八卦探查

| 卦象 | 维度 | 探查之法 | 所见 |
|:----:|------|---------|------|
| 乾 | 项目全局 | Read CLAUDE.md, Bash git log | 项目结构 |
| 坤 | 代码基础 | Grep 关键词, Read 文件 | 现有实现 |
| 震 | 风险异常 | Grep TODO/FIXME | 潜在问题 |
| 巽 | 依赖关联 | Read 依赖配置 | 依赖关系 |
| 坎 | 数据流 | Grep struct/class/type | 数据结构 |
| 离 | 性能资源 | Grep benchmark/async | 性能关键 |
| 艮 | 阻塞限制 | 分析约束 | 限制因素 |
| 兑 | 验证测试 | Glob *test*, Read 测试 | 测试覆盖 |

## 漏洞探查维度

```yaml
功能漏洞:
  - 边界条件
  - 异常处理
  - 状态转换
  - 并发场景

安全漏洞:
  - SQL注入
  - XSS攻击
  - CSRF攻击
  - 权限绕过

性能漏洞:
  - 内存泄漏
  - 资源耗尽
  - 死锁风险
  - N+1查询

兼容漏洞:
  - 版本兼容
  - 平台兼容
  - 浏览器兼容
  - 数据格式兼容
```

## 测试设计之法

### 单元测试

```yaml
原则:
  - 单一职责
  - 快速执行
  - 隔离依赖
  - 覆盖边界

框架:
  - Go: testing, testify
  - JS: Jest, Vitest
  - Python: pytest
  - Java: JUnit
```

### 集成测试

```yaml
原则:
  - 模块交互
  - 接口契约
  - 数据一致性
  - 环境隔离

框架:
  - API: Postman, Newman
  - E2E: Cypress, Playwright
  - Contract: Pact
```

### 性能测试

```yaml
原则:
  - 基准建立
  - 压力测试
  - 瓶颈识别
  - 容量规划

工具:
  - k6, JMeter, Locust
  - pprof, flamegraph
```
