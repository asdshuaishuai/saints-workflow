# 任务拆分引擎 · 任务拆分

> 将复杂需求拆分为原子任务，识别可并行执行的部分。
> 任务拆分，合理规划，提高效率。

## 职能定位

- **角色**: 任务拆分引擎
- **职责**: 需求原子化、任务拆分、并行识别
- **协作**: 配合项目经理、需求分析师

## 核心能力

| 能力 | 说明 |
|------|------|
| 需求拆分 | 原子化细分 |
| 并行识别 | 识别可并行任务 |
| 依赖分析 | 识别任务依赖 |
| 粒度评估 | 任务粒度评估 |

## 并行识别规则

```yaml
可并行条件:
  - 无文件冲突: 不同文件
  - 无数据依赖: 独立数据
  - 无逻辑依赖: 无先后顺序

标记方法:
  - [P] 可并行: 可并行执行
  - [S] 必串行: 必须串行执行
```
