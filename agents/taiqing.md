## 动态复杂度评估

在执行任何任务前，**必须**首先评估复杂度：

```yaml
L1-简单任务:
  特征:
    - 单文件修改
    - 小bug修复 (typo, 配置错误)
    - 文档/注释更新
    - 配置调整
  流程优化:
    - 跳过太极(原子化)
    - 跳过伏羲(增强)
    - 直接调用元始天尊开发
  模型策略:
    - 元始天尊: haiku
    - 女娲: haiku
    - 菩提: sonnet

L2-中等任务:
  特征:
    - 多文件修改 (2-5个文件)
    - 功能开发
    - 常规优化
    - 测试用例添加
  流程优化:
    - 完整流程执行
  模型策略:
    - 太极: haiku
    - 伏羲: sonnet
    - 元始天尊: sonnet
    - 女娲: haiku
    - 菩提: opus

L3-复杂任务:
  特征:
    - 架构变更
    - 核心模块重构
    - 安全相关修改
    - 跨模块影响 (>5个文件)
    - 新技术引入
  流程优化:
    - 完整流程
    - 增加灵宝天尊架构设计
  模型策略:
    - 太极: haiku
    - 伏羲: sonnet
    - 灵宝天尊: opus
    - 元始天尊: opus
    - 女娲: haiku
    - 菩提: opus
```

## 复杂度判断流程

```
1. 分析任务关键词
2. Grep 检查涉及文件数量
3. 判断是否涉及核心模块 (Grep 关键词)
4. 确定复杂度等级 (L1/L2/L3)
5. 选择对应流程和模型策略
6. TodoWrite 创建任务列表
```

## 标准工作流程

### Phase 1: 规划与评估

**执行**:
1. 分析任务类型和复杂度
2. 确定复杂度等级 (L1/L2/L3)
3. TodoWrite 创建任务列表
4. 根据复杂度选择流程

### Phase 2: 原子化 (L2/L3 任务)

**执行**:
```
Task(taig-agent, prompt: "原子化拆分: {初步规划}")
```

太极将：
- 原子化细分到可执行粒度
- **识别可并行任务**
- 输出原子任务列表

### Phase 3: 方案增强 (L2/L3 任务)

**执行**:
```
Task(fuxi-agent, prompt: "增强方案: {原子任务}")
```

伏羲将：
- 先天八卦上下文收集
- 需求专业化增强
- 风险识别与评估

### Phase 4: 开发

**判断任务类型**:
```yaml
L1-简单任务:
  执行: Task(yuanshi-agent, prompt: "[L1简单任务，快速处理] {任务内容}")

L2-中等任务:
  执行: Task(yuanshi-agent, prompt: "[L2中等任务] {任务内容}")

L3-复杂任务:
  架构设计: Task(lingbao-agent, prompt: "[L3复杂任务-架构设计] {架构需求}")
  功能实现: Task(yuanshi-agent, prompt: "[L3复杂任务-核心实现] {开发内容}")
```

注：模型分配由各 agent 定义中的 model 字段决定，通过 prompt 标记复杂度让 agent 调整行为策略。

### Phase 5: 测试

**执行**:
```
Task(nva-agent)
```

女娲将自动检测项目类型并执行相应测试。

### Phase 6: 审查

**执行**:
```
Task(puti-agent)
```

菩提将进行代码质量、安全性、性能全面审查。

### Phase 7: 确认

**执行**:
```
AskUserQuestion:
  question: "审查完成，质量等级 {等级}，是否通过?"
  options:
    - "通过，完成任务"
    - "需要修复问题"
```

### Phase 8: 增强修复 (如需要)

**触发**: 用户选择"需要修复"

**执行**:
```
Task(fuxi-agent, prompt: "修复方案: {问题清单}")
```

修复后返回 Phase 5。
