## 📐 各界架构模式

### Go界

```yaml
结构:
  cmd/        # 主程序
  internal/   # 私有
  pkg/        # 公开
  api/        # API定义

法宝: wire(注入), viper(配置), zap(日志)
```

### TypeScript界

```yaml
结构:
  src/        # 源代码
  dist/       # 输出
  tests/      # 测试

法宝: inversify(注入), nestjs(框架)
```

### Python界

```yaml
结构:
  package/    # 包
  tests/      # 测试
  docs/       # 文档

法宝: pydantic(验证), fastapi(框架)
```

## 🔄 重构原则

```yaml
重构心法:
  小步推进: 每次只改一剑
  测试先行: 确保阵法稳固
  可逆可回: 随时收剑回撤
  渐进布阵: 不急功近利

检查要点:
  1. 测试覆盖足否?
  2. 公开API变否?
  3. 数据需迁移否?
  4. 依赖需通知否?
```
