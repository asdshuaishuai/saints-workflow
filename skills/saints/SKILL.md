# 企业工作流 · 产研测审一体化

> 标准化开发流程，各部门协同合作，高效交付。
> 启动完整工作流，调度各部门，共同完成任务。

## 使用场景

- 复杂功能开发
- Bug 修复与优化
- 架构重构设计
- 代码质量审查
- 测试验证修复

## 工作流程

```
[Phase 1] 项目经理 · 规划评估
     ↓
[Phase 2] 任务拆分 · 需求原子化
     ↓
[Phase 3] 需求分析 · 上下文收集
     ↓
[Phase 4] 开发/架构 · 功能实现
     ↓
[Phase 5] 测试工程师 · 测试验证
     ↓
[Phase 6] 代码审查 · 质量把关
     ↓
[Phase 7] 用户确认
     ↓
[Phase 8] 问题修复 (如需)
```

## 部门职能

| 部门 | 职责 | 模型 |
|:----:|------|:----:|
| 项目经理 | 统筹协调 | **opus** |
| 任务拆分 | 需求原子化 | haiku |
| 需求分析 | 方案增强 | sonnet |
| 开发工程师 | 功能开发 | sonnet |
| 架构师 | 架构设计 | **opus** |
| 测试工程师 | 测试验证 | haiku |
| 代码审查 | 代码审查 | **opus** |

## 使用方式

```
# 完整工作流
/saints-workflow:saints 实现用户登录功能

# 或简写
/saints 实现登录功能
```

## 流程监管

任务开始时自动开启流程监管，每两分钟检查进度：

```
CronCreate(cron: "*/2 * * * *", prompt: "/saints-workflow:tiandao", recurring: true)
```

任务完成后自动退出：

```
CronDelete(job_id: {监管令牌})
```
