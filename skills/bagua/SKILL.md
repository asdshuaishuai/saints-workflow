# 上下文分析器 (Bagua)

上下文分析器是一个信息收集技能，从8个维度全面分析项目状态。

## 使用场景

- 需求分析前收集上下文
- 架构设计前了解现状
- 代码审查前的准备
- 问题诊断时的信息收集

## 八维度分析框架

```
                全局视角 (项目概况)
                      │
    数据视角 ─────────┼───────── 性能视角
                      │
    风险视角 ─────────┼───────── 依赖视角
                      │
    限制视角 ─────────┼───────── 测试视角
                      │
                基础视角 (代码现状)
```

## 收集动作

| 维度 | 分析内容 | 收集命令 | 目的 |
|:----:|---------|---------|------|
| 全局 | 项目概况 | `Read CLAUDE.md/README`<br>`Glob **/*`<br>`Bash git log --oneline -10` | 了解项目结构、历史 |
| 基础 | 代码现状 | `Grep "关键词"`<br>`Read 相关文件` | 理解现有实现 |
| 风险 | 潜在问题 | `Grep "TODO\|FIXME\|XXX\|HACK"` | 发现潜在问题 |
| 依赖 | 依赖关系 | `Glob *依赖文件*`<br>`Read 依赖配置` | 了解依赖关系 |
| 数据 | 数据结构 | `Grep "类型定义\|接口\|数据结构"`<br>`Glob *model/entity/schema*` | 理解数据结构 |
| 性能 | 性能关键 | `Grep "性能相关代码"`<br>`Glob *test/benchmark*` | 评估性能状况 |
| 限制 | 约束条件 | 分析约束条件、技术债务 | 识别限制因素 |
| 测试 | 测试覆盖 | `Glob *test/spec*`<br>`Read 测试文件` | 了解测试覆盖 |

**常见依赖文件**: go.mod, package.json, requirements.txt, Cargo.toml, pom.xml, Gemfile, composer.json, build.gradle 等

## 输出格式

```markdown
## 上下文分析报告

### 全局视角 - 项目概况
- 项目类型: {type}
- 主要技术栈: {stack}
- 近期变更: {changes}

### 基础视角 - 代码现状
- 相关文件: {files}
- 核心模块: {modules}

### 风险视角 - 潜在问题
- 待处理: {todos}
- 已知问题: {issues}

### 依赖视角 - 依赖关系
- 直接依赖: {deps}
- 依赖风险: {risks}

### 数据视角 - 数据结构
- 数据结构: {structs}
- 接口定义: {interfaces}

### 性能视角 - 性能关键
- 性能关键点: {points}
- 资源消耗: {resources}

### 限制视角 - 约束条件
- 技术约束: {constraints}
- 待解决: {blocks}

### 测试视角 - 测试覆盖
- 测试覆盖: {coverage}
- 测试状态: {status}

### 综合评估
{summary}
```

## 使用方式

```
# 直接调用技能
/bagua 收集用户认证模块的上下文

# 或在需求分析中自动调用
```
