# 聖人ワークフロー (Saints Workflow)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**言語**: [中文](README.md) | [English](README_EN.md) | [日本語](README_JA.md)

> *混沌初めて分かれ、天地は暗し。太古の聖人、神威を現す。*

中国神話「封神演義」に基づくClaudeエージェント多役割協作システム — ソフトウェア開発の各段階を神話体系の専門家たちの分業にマッピング

---

## プロジェクトの特徴

**独創的な多役割エージェントシステム**の特徴：

- **20+ 専門的エージェント** - 要件分析からコードレビューまでの完全な役割体系
- **5つコア技能** - 先天八卦、神阵演武、封神天榜、先天灵宝、聖人ワークフロー
- **阐截対抗モード** - 正規開発(阐教)vs革新的提案(截教)の双線追跡評審
- **西方双審機制** - コード審査(准提) + ビジネス審査(接引)の二重チェック
- **動的タスクルーティング** - タスク複雑度に応じたエージェント自動調度

### ワークフロー

```
[1] 要件分析    [2] 原子分解    [3] 対抗開発    [4] テスト検査

┌────────────────────────────────────────────┐
│     西方二審(コード+ビジネス)              │

└────────────────────────────────────────────┘
```

## 使用ガイド

### 完全ワークフロー

```bash
/saints-workflow:saints ユーザーログイン機能を実装
```

自動実行内容:
1. **太上老君** が要件を分析
2. **太極** が原子タスクに分解
3. **阐教** が正規案、**截教** が革新案を提案
4. **伏羲** が8次元欠陥探査
5. **女娲** が問題を修復
6. **准提** がコード審査、**接引** がビジネス審査
7. **天道** が最終確認

### 単一エージェントコマンド

| 聖人 | コマンド | 用途 | 例 |
|------|---------|------|-----|
| 太上老君 | `/saints-workflow:taishang` | 要件分析 | 新機能要件分析 |
| 太極 | `/saints-workflow:taig` | 原子分解 | 大型リファクタリング分解 |
| 元始天尊 | `/saints-workflow:yuanshi` | 正規開発 | 標準実装 |
| 通天教主 | `/saints-workflow:tongtian` | 対抗提案 | 革新的なアプローチ提案 |

## アーキテクチャ

### エージェント体系
- **20+ 専門的エージェント** - 各エージェントが専門的役割に対応、明確な職責とツールセット
- **3層モデル配置**: opus(複雑決定) > sonnet(標準開発) > haiku(簡単タスク)
- **役割協力** - Taskコマンドを通じた相互協作

### スキル体系
- **5つコア技能** - ワークフロー全体を支援
- **自動トリガー** - 適切なフェーズで自動起動
- **段階的設計** - グローバル洞察から詳細チェックまで
