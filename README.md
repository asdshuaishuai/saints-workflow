# 圣人工作流 (Saints Workflow)

基于道教神话的智能开发协作系统

---

## 概述

圣人工作流是一个完整的开发协作系统，包含7个阶段的工作流程：

```
[1] 规划 → [2] 拆分增强 → [3] 开发 → [4] 测试 → [5] 审查 → [6] 确认 → [7] 增强修复
```

### 核心特性

- **动态模型分配**: 根据任务复杂度自动选择模型 (opus/sonnet/haiku)
- **并行任务识别**: 自动识别可并行执行的任务
- **智能流程简化**: 简单任务自动跳过不必要阶段

## 动态复杂度评估

```yaml
L1-简单任务:
  特征: 单文件、小bug、配置修改
  流程: 直接开发 → 测试 → 完成
  跳过: 太极原子化、伏羲增强
  模型: haiku 为主

L2-中等任务:
  特征: 多文件(2-5)、功能开发
  流程: 完整流程
  模型: 混合使用

L3-复杂任务:
  特征: 架构变更、核心重构、跨模块
  流程: 完整流程 + 灵宝天尊架构设计
  模型: 关键角色使用 opus
```

## 安装

### 方法1: 项目级安装

```bash
# 复制到项目 .claude/plugins/ 目录
cp -r saints-workflow /path/to/project/.claude/plugins/
```

### 方法2: 全局安装

```bash
# 复制到全局 plugins 目录
cp -r saints-workflow ~/.claude/plugins/
```

## 钩子功能

### pre-edit-check.sh
- 保护关键依赖文件 (go.mod, package.json 等)
- 文件锁检测 (防止并发冲突)
- 自动备份配置文件

### post-edit-verify.sh
- 快速语法检查 (不完整编译)
- 变更历史记录
- 文件大小异常检测
- 测试文件关联提示
