# 圣人工作流 (Saints Workflow)

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**语言 / Languages**: [中文](README.md) | [English](README_EN.md) | [日本語](README_JA.md)

> *混沌初开，天地玄黄。洪荒众圣，各显神通。*

基于洪荒封神演义的 Claude Agent 多角色协作系统 —— 将软件开发的各个环节映射为神话体系中诸圣的分工协作。

---

## 项目特色

这是一个**创意型 Agent 多角色系统**，具有以下特点：

- **20+ 专业 Agent** - 从需求分析到代码审查的完整角色体系
- **5 大核心技能** - 先天八卦、神阵演武、封神天榜、先天灵宝、圣人工作流
- **阐截对抗模式** - 正面开发(阐教) vs 激进方案(截教)的双线程评审
- **西方双审机制** - 代码审查(准提) + 业务审查(接引) 的双重把关
- **动态任务路由** - 根据任务复杂度自动调度最适合的 Agent

## 5 大核心技能

### 1. 先天八卦 (Bagua)

8 维度上下文智能收集，快速理解项目现状：

```yaml
☰ 乾 - 项目全局      # 项目结构、技术栈、历史变更
☷ 坤 - 代码基础      # 相关文件、核心模块
☳ 震 - 风险异常      # TODO/FIXME/已知问题
☴ 巽 - 依赖关联      # 依赖关系、版本风险
☵ 坎 - 数据流        # 数据结构、接口定义
☲ 离 - 性能资源      # 性能关键点、资源消耗
☶ 艮 - 阻塞限制      # 技术约束、待解决项
☱ 兑 - 验证测试      # 测试覆盖、测试状态
```

**使用**: `/bagua [目标模块]`

### 2. 神阵演武 (Zhenfa)

5 大阵法轮转，应对不同规模的开发任务：

| 阵法 | 阵主 | 适用场景 | 功能 |
|------|------|---------|------|
| 两仪微尘阵 | 太上老君 | 需求初分 | 阴阳分离、原子化 |
| 万仙阵 | 通天教主 | L3 大任务 | 多 Agent 并行协作 |
| 诛仙阵 | 通天教主 | 关键决策 | 四圣破阵决大事 |
| 九曲黄河阵 | 三霄娘娘 | 综合测试 | 9 层递进式测试 |
| 十绝阵 | 吕岳 | 全面扫描 | 10 重深度检查 |

### 3. 封神天榜 (Fengshen)

三榜制度，全程追踪任务状态：

```yaml
功德榜:  # 已完成任务、功德值统计、排行榜
轮回榜:  # 待修复 Bug、优先级、责任人
因果榜:  # 技术债务追踪、影响范围
```

### 4. 先天灵宝 (Fabao)

开发工具能力映射为法宝，每个法宝配置具体的检查规则：

| 法宝 | 功能 | 持宝人 | 示例 |
|------|------|--------|------|
| 混元金斗 | 代码质量扫描 | 云霄娘娘 | 复杂度、重复代码 |
| 金蛟剪 | 安全漏洞检测 | 琼霄娘娘 | SQL 注入、XSS |
| 缚龙索 | 前端性能监控 | 碧霄娘娘 | 加载时间、内存泄漏 |
| 定海珠 | 数据库监控 | 赵公明 | 慢查询、索引效率 |
| 多宝如意 | 架构验证 | 多宝道人 | 分层、依赖关系 |

### 5. 圣人工作流 (Saints)

完整的多阶段开发流程，自动调度所有圣人和技能。

## 使用指南

### 完整工作流

```bash
/saints-workflow:saints 实现用户登录功能
```

自动执行完整流程：
1. **太上老君** 分析需求、评估复杂度
2. **太极** 原子化拆分成子任务
3. **阐教** 提出正面方案、**截教** 提出激进方案
4. **伏羲** 进行 8 维度漏洞探查
5. **女娲** 修复发现的问题
6. **准提** 进行代码审查、**接引** 进行业务审查
7. **天道** 最终确认和总结

## 架构设计

### Agent 体系

- **20+ 专业 Agent** - 每个 Agent 对应一个专业角色，有明确的职责和工具集
- **3 层模型分配**：opus (复杂决策) > sonnet (标准开发) > haiku (简单任务)
- **角色协作** - 通过 Task 指令相互协作，形成完整的工作流

### Skill 体系

- **5 大核心技能** - 支撑整个工作流系统
- **自动触发** - 在合适的流程阶段自动激活
- **递进式设计** - 从全局洞察到细节检查

### Command 入口

- **saints.md** - 完整工作流入口
- **其他 Agent 命令** - 单个圣人的快速调用
