## 启动流程

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   启动项目经理，统筹项目
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**第一步：开启流程监管**

```
CronCreate(cron: "*/2 * * * *", prompt: "/saints-workflow:monitor", recurring: true)
```

流程监管器每两分钟自动检查进度，发现停滞、迷茫、循环等问题时，及时干预纠正。

**第二步：启动项目经理**

```
Task(pm, prompt: "【监管已开启】执行企业工作流: $ARGUMENTS")
```

| 部门 | 角色 | 职责 | 模型 |
|:----:|:----:|------|:----:|
| 项目经理 | 流程控制 | 统筹协调，把控进度 | **opus** |
| 任务拆分 | 需求分析 | 任务原子化，并行识别 | haiku |
| 需求分析 | 业务分析 | 上下文收集，方案增强 | sonnet |
| 开发工程师 | 核心开发 | 功能开发，代码实现 | sonnet |
| 架构师 | 技术架构 | 架构设计，技术选型 | **opus** |
| 测试工程师 | 质量保证 | 测试验证，问题修复 | haiku |
| 代码审查 | 质量把关 | 代码审查，安全检查 | **opus** |
| 流程监管 | 进度监控 | 流程监管，异常干预 | sonnet |

## 工作流程

```
┌─────────────────────────────────────────────┐
│  【需求接收】用户下达任务                    │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 1】项目经理 · 规划评估              │
│  评估任务复杂度，确定 L1/L2/L3 流程         │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 2】任务拆分 · 需求原子化            │
│  任务拆分，并行识别，依赖分析               │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 3】需求分析 · 上下文收集            │
│  八维度分析，方案增强，风险评估             │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 4】开发/架构 · 功能实现             │
│  L3加架构设计，各部门并行开发               │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 5】测试工程师 · 测试验证            │
│  测试构建，问题修复，质量验证               │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 6】代码审查 · 质量把关              │
│  代码审查，安全检查，质量评级               │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 7】用户确认                         │
│  请示用户，是否通过交付                      │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【Phase 8】问题修复 (如需)                  │
│  返工修复，回到 Phase 5                     │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  【任务完成】监管退出                        │
│  CronDelete 解除定时监管                    │
│  各部门归位，流程结束                       │
└─────────────────┘
```
