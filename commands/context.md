# 上下文分析器 · 项目分析

调用 bagua-skill 从八维度收集项目上下文。

## 目标: $ARGUMENTS

## 执行

调用上下文分析技能收集信息：

```
Skill(bagua-skill, args: "$ARGUMENTS")
```

## 分析维度

| 维度 | 分析内容 | 目的 |
|:----:|---------|------|
| 全局 | 项目概况 | 项目结构、技术栈、历史变更 |
| 基础 | 代码现状 | 相关文件、核心模块 |
| 风险 | 潜在问题 | TODO/FIXME/已知问题 |
| 依赖 | 依赖关系 | 依赖关系、版本风险 |
| 数据 | 数据结构 | 数据结构、接口定义 |
| 性能 | 性能关键 | 性能关键点、资源消耗 |
| 限制 | 约束条件 | 技术约束、待解决项 |
| 测试 | 测试覆盖 | 测试覆盖、测试状态 |

## 使用示例

```bash
# 收集整个项目上下文
/bagua

# 收集特定模块上下文
/bagua 用户认证模块

# 功能开发前使用
/bagua 支付功能
```
